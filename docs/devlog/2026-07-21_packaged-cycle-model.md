# Inventory Tracker Devlog — 포장 개수와 개봉·소진 모델 분리

- **Date**: 2026-07-21
- **Repository**: `mworkroom/inventory-tracker`

## 발견한 문제

기존 `cycle` 방식은 현재 재고를 g·ml 총량으로 저장하면서 동시에 한 제품의 개봉→소진 주기를 측정했다.

예:

```text
코코넛 오일 1개 = 1,600ml
사용 중 제품 잔량 = 300ml
미개봉 제품 = 1개
총 용량 = 1,900ml
```

기존 동작에서는 입고 후 `다 씀`을 누르면 제품 전체 용량 1,600ml를 차감해 300ml만 남았다. 실제로는 미개봉 1개가 남아 있으므로 1통이 남아야 했다.

원인은 다음 두 개념을 하나의 값으로 관리한 것이었다.

```text
보유한 포장 개수
현재 사용 중인 포장의 잔량
```

## 새 기록 방식

제품 등록 시 세 가지 실사용 방식을 선택한다.

```text
count
- 개수 직접 차감
- 예: 닭안심 5인분

cycle
- 재고는 통·병·봉 개수로 관리
- 한 제품은 개봉→소진으로 사용 기간 측정
- 예: 코코넛 오일 2통, 토너 3병, 구연산 1봉

capacity
- 남은 g·ml를 직접 관리하고 사용할 때 용량 차감
- 예: 보관통 속 가루 800g
```

## cycle 제품의 새 의미

```text
current_quantity
= 사용 중 제품을 포함한 현재 총 포장 개수

active_remaining_quantity
= 현재 사용 중인 제품 하나의 남은 g·ml

active_opened_on
= 현재 제품의 개봉일
```

코코넛 오일 예:

```text
현재 재고: 2통
현재 제품: 4/12 개봉 · 약 300ml 남음
미개봉: 1통
```

동작:

```text
구매 기록
→ 주문일과 구매처만 저장, 재고 변화 없음

입고 +1통
→ 현재 총 포장 개수만 1 증가

다 씀
→ 현재 총 포장 개수에서 1 차감
→ 개봉일부터 소진일까지 사용 주기 저장

새 제품 개봉
→ 재고 개수 변화 없음
→ 새 개봉일과 시작 잔량 저장

현재 잔량
→ 사용 중 제품의 g·ml만 수정
→ 전체 포장 개수 변화 없음
```

## 기존 데이터 변환

기존 cycle 제품의 총용량을 제품 1개 용량으로 나눠 자동 변환했다.

```text
구연산 2,500g / 5,000g
→ 현재 1개 + 현재 제품 잔량 2,500g

스빈또 수세미 100g / 200g
→ 현재 1개 + 현재 제품 잔량 100g

코코넛 오일 1,600ml / 1,600ml
→ 현재 1개
```

기존 구매 기록과 완료된 사용 주기는 유지했다. 기존 최초 재고 이벤트의 수량도 새 포장 개수에 맞게 변환했다.

## UI 변경

- 제품 추가 화면의 방식 선택을 3개로 재구성
- `개봉 → 소진` 방식에서 재고 단위와 제품 1개 용량을 분리 입력
- 카드에 현재 포장 개수와 현재 제품 잔량을 별도 표시
- `현재 잔량` 빠른 기록 버튼 추가
- 입고·소진 다이얼로그 문구를 포장 개수 기준으로 수정
- `다 씀` 실행 전 감소 후 개수를 명시
- cycle 제품 카드에 입고와 소진의 의미 안내 추가

## 검증

### 포장 제품 흐름

```text
현재 1통
→ 개봉 잔량 300ml 기록
→ 입고 +1통
→ 재고 2통
→ 다 씀
→ 재고 1통
```

확인 결과:

```text
입고 후: 2통
소진 후: 1통
완료 사용 주기: 1회
현재 제품 상태: 초기화
```

### 용량 직접 차감

```text
현재 800g
→ 125.5g 사용
→ 674.5g
```

소수 용량 차감이 정상 동작하는 것을 확인했다.
